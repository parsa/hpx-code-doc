20 ------------------------------
21 network_storage.bat.in
22 ------------------------------
23 This is a windows batch file template which will be filled in using cmake substitution
24 and then copied to your build/scripts directory.
25 The generated script can be invoked to run the benchmark in serial or parallel.
26 It spawns each task on the same node so although you may run 4 copies,
27 they are competing for resources.
28 The windows version is good for testing and debugging on a single node.
29 adding or removing /B for the launch command in the script
30 controls whether each task is launched in a new console window or not
31 example usage (2 tasks)
32 scripts/network_storage.bat 2
33 
34 ------------------------------
35 slurm-test-HPX-storage.sh.in
36 ------------------------------
37 This is a bash script template which will be filled in using cmake substitution
38 and then copied to your build/scripts directory.
39 When executed, the script will loop over a number of parameter combinations
40 and for each create a directory with a name generated from the paramaters, such as
41  hpx-N0002-T02048-t04-ibverbs
42 and inside the dir there will be a "submit-job.bash" script which contains a single
43 slurm job submission. You can manually submit just one job using "sbatch submit-job.bash",
44 or in the root of the scripts folder there will be generated another script
45 which is called "run_jobs.bash". When this script is run, it will loop over
46 the jobs that were created by the first script and submit them all, you
47 can then sit back and wait until they complete.
48 Each job will write its results into "slurm.out" (and errors to slurm.err)
49 in the same subdirectory in which each individual job submission script was created.
50 
51 The network_storage executable will produce several lines of output, but one line
52 contains the condensed information needed by the plotting script.
53 This line begins with the text "CSVData" to indicate comma separated data values.
54 Whilst jobs are running or when they have completed, you can execute a command
55 from the test root dir, such as
56  find . -name slurm.out -exec grep CSV {} \;
57 and a list of results generated from the jobs will be produced.
58 For plotting of results, the output should be directed into a file using
59  find . -name slurm.out -exec grep CSV {} \; >results-bgq-1a-2014-04-01.csv
60 Where you use a file name applicable to your current experiment.
61 The generated file will be suitable for use by the plotting script.
62 
63 Important Note:
64 By default, the slurm jobs generated by the script have the 'exclusive' flag set.
65 This is because the jobs are intended to test BW of the network and it is
66 often desirable to have only a single job running at a time.
67 The 'exclusive' flag only works (under slurm) if all jobs have the same name
68 so in your queue you will see many identical jobs and it will take a long time to drain.
69 If you can afford to run many job simultaneously, the job name may be set more
70 appropriately (details in script) and the exclusive flag removed.
71 
72 ------------------------------
73 plot-results.py
74 ------------------------------
75 This is a python script which takes results generated by the test program
76 as described above and plots a number of graphs for different parameter studies.
77 The results.csv file generated contains BW measurements, timing ,thread, parcelport,
78 block size, etc information for the plots.
79 The python script parses the results and generates arrays (maps) of the data which
80 can be plotted in various ways. The scripts can be invoked as
81  plot-results.py results.csv
82 optional arguments such as the figure size can be found by looking at the script.
83 The output from the script will be a series of svg and png files for each of the plots
84 created.
