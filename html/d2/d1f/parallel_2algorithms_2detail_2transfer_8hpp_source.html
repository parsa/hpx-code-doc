66  // Executes transfer algorithm on the elements in the range [first, last),
67  // to another range beginning at \a dest.
68  //
69  // \note Complexity: Performs exactly \a last - \a first transfer assignments.
70  //
71  //
72  // \tparam Algo The algorithm that is used to transfer the elements.
73  // Should be hpx::parallel::detail::copy or
74  // hpx::parallel::detail::move.
75  // \tparam ExPolicy The type of the execution policy to use (deduced).
76  // It describes the manner in which the execution
77  // of the algorithm may be parallelized and the manner
78  // in which it executes the move assignments.
79  // \tparam FwdIter1 The type of the source iterators used (deduced).
80  // This iterator type must meet the requirements of an
81  // forward iterator.
82  // \tparam FwdIter2 The type of the iterator representing the
83  // destination range (deduced).
84  // This iterator type must meet the requirements of an
85  // output iterator.
86  //
87  // \param policy The execution policy to use for the scheduling of
88  // the iterations.
89  // \param first Refers to the beginning of the sequence of elements
90  // the algorithm will be applied to.
91  // \param last Refers to the end of the sequence of elements the
92  // algorithm will be applied to.
93  // \param dest Refers to the beginning of the destination range.
94  //
95  // \returns The \a transfer algorithm returns a \a hpx::future<FwdIter2> if
96  // the execution policy is of type
97  // \a sequenced_task_policy or
98  // \a parallel_task_policy and
99  // returns \a FwdIter2 otherwise.
100  // The \a move algorithm returns the output iterator to the
