20 // A quick wrapper around an unordered_map with a mutex to ensure two
21 // threads don't simultaneously write or read during write.
22 // Warning: not thread safe to use iterators whilst others are changing the map
23 // obtain a read lock before iterating using the provided mutex and function
24 // map_type::map_read_lock_type read_lock(map.read_write_mutex());
25 // in order to safely iterate over contents and block any writers from gaining access
26 //
