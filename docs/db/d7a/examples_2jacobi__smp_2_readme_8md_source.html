6 This is a version of a Jacobi supposed to run on shared memory machines.
7 It based on the dataflow ideas as presented in this paper:
8 http://dl.acm.org/citation.cfm?id=2467126
9 
10 The example consists of 2 parts, each of the part provides a jacobi smoother
11 implemented in HPX and OpenMP, while the OpenMP variant includes one with
12 static and one with dynamic scheduling policies.
13 
14 The first variant smoothes a regular two-dimensional grid with a simple
15 5 point stencil. The parameters are the number of grid points in one dimension
16 and for the HPX version, a block-size parameter which determines the
17 granularity of the work done. The relevant executables are:
18  * jacobi_hpx
19  * jacobi_omp_static
20  * jacobi_omp_dynamic
21 
22 The second variant performs a dynamic stencil based on the neighborhood
23 given by a sparse matrix. The matrix input format is "Matrix Market".
24 An example matrix can be obtained here:
25 http://www.cise.ufl.edu/research/sparse/matrices/Janna/Serena.html
26 Other matrices from that portal work as well.
27 The relevant executables are:
28  * jacobi_nonuniform_hpx
29  * jacobi_nonuniform_omp_static
30  * jacobi_nonuniform_omp_dynamic
