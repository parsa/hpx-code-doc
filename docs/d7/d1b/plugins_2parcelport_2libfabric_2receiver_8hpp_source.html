28  // The receiver is responsible for handling incoming messages. For that purpose,
29  // it posts receive buffers. Incoming messages can be of two kinds:
30  // 1) An ACK message which has been sent from an rma_receiver, to signal
31  // the sender about the succesful retreival of an incoming message.
32  // 2) An incoming parcel, that consists of an header and an eventually
33  // piggy backed message. If the message is not piggy backed or zero
34  // copy RMA chunks need to be read, a rma_receiver is created to
35  // complete the transfer of the message
