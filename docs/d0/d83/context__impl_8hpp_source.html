38 /*
39  ContextImpl concept documentation.
40  A ContextImpl holds a context plus its stack.
41  - ContextImpl must have constructor with the following signature:
42 
43  template<typename Functor>
44  ContextImpl(Functor f, std::ptrdiff_t stack_size);
45 
46  Preconditions:
47  f is a generic function object (support for function pointers
48  is not required.
49  stack_size is the size of the stack allocated
50  for the context. The stack_size is only an hint. If stack_size is -1
51  it should default to a sensible value.
52  Postconditions:
53  f is bound to this context in its own stack.
54  When the context is activated with swap_context for the first time
55  f is entered.
56 
57  - ContextImpl destructor must properly dispose of the stack and perform
58  any other clean up action required.
59 
60  - ContextImpl is not required to be DefaultConstructible nor Copyable.
61 
62  - ContextImpl must expose the following typedef:
63 
64  typedef unspecified-type context_impl_base;
65 
66  ContextImpl must be convertible to context_impl_base.
67  context_impl_base must have conform to the ContextImplBase concept:
68 
69  - DefaultConstructible. A default constructed ContextImplBase is in
70  an initialized state.
71  A ContextImpl is an initialized ContextImplBase.
72 
73  - ContextImplBase is Copyable. A copy of a ContextImplBase holds
74  The same information as the original. Once a ContextImplBase
75  is used as an argument to swap_context, all its copies become stale.
76  (that is, only one copy of ContextImplBase can be used).
77  A ContextImpl cannot be sliced by copying it to a ContextImplBase.
78 
79  - void swap_context(ContextImplBase& from,
80  const ContextImplBase& to)
81 
82  This free function is called unqualified (via ADL).
83  Preconditions:
84  Its 'to' argument must be an initialized ContextImplBase.
85  Its 'from' argument may be an uninitialized ContextImplBase that
86  will be initialized by a swap_context.
87  Postconditions:
88  The current context is saved in the 'from' context, and the
89  'to' context is restored.
90  It is undefined behavior if the 'to' argument is an invalid
91  (uninitialized) swap context.
92 
93  A ContextImplBase is meant to be used when an empty temporary
94  context is needed to store the current context before
95  restoring a ContextImpl and no current context is
96  available. It could be possible to simply have ContextImpl
97  default constructible, but on destruction it would need to
98  check if a stack has been allocated and would slow down the
99  fast invocation path. Also a stack-full context could not be
100  make copyable.
101 */
