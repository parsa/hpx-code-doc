Detailed Description

The parcelhandler is the representation of the parcelset inside a locality. It is built on top of a single parcelport. Several parcel-handlers may be connected to a single parcelport.

)

A parcel is submitted for transport at the source locality site to the parcel set of the locality with the put-parcel command

Note
The function put_parcel() is asynchronous, the provided function or function object gets invoked on completion of the send operation or on any error.
Parameters
p[in, out] A reference to the parcel to send. The parcel p will be modified in place, as it will get set the resolved destination address and parcel id (if not already set).
f[in] A function object to be invoked on successful completion or on errors. The signature of this function object is expected to be:
void f (boost::system::error_code const& err, std::size_t );
            where \a err is the status code of the operation and
                  \a size is the number of successfully
                         transferred bytes.
